# R7 零基础双语 Java Lesson 教学质量标准

> 版本: R7.RECOVERY-0
> 适用: `feature/r7-java-course-recovery`
> 参考基线: 43724d8 (R7.0A.1)
>
> 本标准不依赖任何批量改写脚本的输出，仅基于"高质量零基础双语 Java 教学"的一贯标准。

---

## 0. 核心原则

1. **每节必须回答一个问题**：为什么学生需要看这一节？这一节让他们能做出什么以前做不出的判断？
2. **日文不翻译中文，中文不直译日文**：两种语言各自独立为目标读者补足理解台阶。
3. **代码行为比代码文字更重要**：学生改一个参数后看到的输出变化，是他们真正学到的东西。
4. **错误教学比无教学更危险**：如果 commonMistakes 只说"注意别写错"，那就是无效教学。
5. **教材页码是元数据，不是教学内容**：学生不需要知道"教材第X页"，他们需要知道"这个概念在代码的哪里起作用"。

---

## 1. 高质量 Lesson 的 8 个判断维度

对每一节 (lesson)，以下 8 个维度必须可独立验证：

| # | 维度 | 验证问题 | 反例 |
|---|---|---|---|
| 1 | 本节概念 | 本节想让学生理解哪一个具体概念？概念边界在哪里？ | "本节学习 Java 语言" — 范围过大 |
| 2 | 为什么需要 | 这个概念在写 Java 程序时解决什么问题？不用它会怎样？ | 只写"这是重要的概念" — 没有给出情境 |
| 3 | 初学者心智模型 | 零基础学生应该在大脑中形成什么样的画面来理解这个概念？ | 直接跳到术语定义 — 没有给画面 |
| 4 | 代码的真实运行行为 | 示例代码的运行行为是否聚焦本节概念？学生能否改一行、看一行结果？ | 示例包含 5 个不相关的概念 — 模糊焦点 |
| 5 | 输入→处理→输出 | 学生能否在代码中指出输入在哪里、处理在哪里、输出在哪里？ | 只展示代码和输出，不解释中间映射 |
| 6 | 本节独有易错点 | commonMistakes 是否描述了本节特有的错误、错误原因、错误后果？ | "注意分号" — 太通用，不是本节特有 |
| 7 | 动手任务的具体观察目标 | handson 是否告诉学生"改什么、看什么、应该看到什么"？ | "自己改一改代码试试" — 没有具体目标 |
| 8 | 与下一节的知识衔接 | bridge 是否精确指向下一节的前置知识？学生对下一节有可验证的准备？ | "下一节继续学习 Java" — 泛泛而谈 |

## 2. 字段级质量标准

### 2.1 title (ja+zh)
- 日文标题：使用日本 IT 教科书的标准术语，简洁准确
- 中文标题：使用中国计算机教育的通用术语，提供日文对照
- **允许**：日文中出现日文汉字和片假名术语
- **禁止**：中文标题直接用日文汉字不加翻译

### 2.2 blocks (5 个教学块)

| block type | 设计意图 | 高质量标准 |
|---|---|---|
| learning-goal | 本节目标 | 一句话说清本节学什么，不要只是"教材X页的XX" |
| mechanic | 代码机制 | 解释代码中关键行的执行机制，不是"教材X页的XX" |
| beginner-note | 零基础阅读顺序 | 给初学者的阅读切入点，从哪一行开始看、按什么顺序 |
| pitfall | 易错点 | 本节特有的错误模式，不是通用编程错误 |
| practice-prep | 动手前确认 | 引导学生先预测再修改，建立"预测→验证→修正"循环 |

**统一要求**：
- 每个 block 的开头不得出现"教材Xページ"或"教材第X页"作为前缀。页码追踪应使用 `sourceRef` 元数据字段。
- 日文 block 应使用自然、准确、适合日本 IT 学校初学者的日语
- 中文 block 不是日文直译，而是补充中文读者特有的理解台阶

### 2.3 codeExamples
- 每个示例必须能被 `javac` 编译、`java` 运行
- className 使用 `JavaR7C{chapter}S{section}` 格式（如 `JavaR7C01S001`）
- 示例代码聚焦本节单一概念，不引入额外未解释的概念
- `expectedOutput` 必须与真实运行结果一致
- `lineNotes` 标注本节概念相关的关键行
- **禁止**在 expectedOutput 中包含 `lesson=xxx` — 这是元数据，不是教学内容
- **禁止**使用 `dummyStream` 或非教学目的的 `System.gc()`

### 2.4 commonMistakes
每条必须包含：
1. **错误行为** — 学生具体做了什么错误操作
2. **错误原因** — 为什么会产生这个错误（认知层面）
3. **错误后果** — 错误会导致什么结果（编译器报错？运行时异常？逻辑错误？）

**反例**（仅术语替换的模板）："只记住'XX'这个词，不确认代码中的位置" — 适用于任何概念，不是本节特有。

### 2.5 handson
必须包含：
1. 具体操作指令（改哪一行、改什么）
2. 预期观察目标（改动后应该看到什么变化）
3. 操作的安全范围（不会导致系统级问题的修改）

**反例**："改一改代码看看结果" — 没有具体目标。

### 2.6 nextLessonBridge
必须包含：
1. 下一节的准确名称
2. 本节知识在下一节的哪个环节会被使用
3. 学生应该带着什么确认问题进入下一节

### 2.7 terms
- `en`：Java 标准英文术语
- `ja`：日本 IT 教育界通用日文术语
- `zh`：中国计算机教育界通用中文术语
- `explanationJa/explanationZh`：不是词典定义，而是在本节代码中的实际含义

## 3. 重复与递进的合理边界

### 3.1 允许重复（不降级标记）

| 类别 | 示例 | 理由 |
|---|---|---|
| 固定栏目标题 | "この節のねらい"、"从代码看重点" | UI 一致性需要 |
| 基础安全提醒 | "编译前请确认文件名与类名一致" | 安全基线 |
| 相邻节的知识衔接 | 前节的 summary 与后节的 prerequisites 有交叉 | 认知连续性 |
| 同一概念链的渐进代码 | ch03-lesson-001 (if) → ch03-lesson-002 (if-else) 共享基本结构 | 递进式教学 |
| Java 固有术语 | "main メソッド"、"public class" | 语言规范 |
| 双语教学的必要句式 | "这个小例子不是教材原文复制" | 防止抄袭教材的法律声明 |

### 3.2 必须识别为风险

| 风险类别 | 检测方法 | 分类标记 |
|---|---|---|
| 核心解释完全重复 | 规范化后 Jaccard >= 0.95，且非相邻节 | LIKELY_TEMPLATE_REPLICATION |
| 代码行为完全相同 | 规范化代码（去类名/变量名/注释/字面量/日志）后完全一致 | CODE_BEHAVIOR_DUPLICATE |
| commonMistakes 仅替换术语 | "只记住'XX'这个词"模板 | LIKELY_TEMPLATE_REPLICATION |
| handson 仅替换标题 | "在'XX'这一节修改一处"模板 | LIKELY_TEMPLATE_REPLICATION |
| 页码混入正文 | blocks/handson/commonMistakes 中出现"教材Xページ" | LEARNER_VISIBLE_METADATA_NOISE |
| lessonId 混入输出 | expectedOutput 中包含 `lesson=xxx` | LEARNER_VISIBLE_METADATA_NOISE |

## 4. 8 个黄金样本

以下 8 节被选为后续内容改写的质量基准。选择标准：覆盖 intro 和 practice 阶段、覆盖不同类型的概念（编译/变量/分支/方法/构造/继承/实战）、每节均有完整的 5 个 block + code + mistakes + handson + bridge 结构。

### GS1: intro-ch01-lesson-001 — プログラムとは (Java 编译 / main)

- **章节**: java-ch01, order 1/13
- **选择理由**: 全课程的第一个 lesson，定义了零基础学生的入口体验。教学主题是"程序是什么"而非"怎么写代码"。
- **当前质量观察**: 5 个 block 均以"教材4ページ"开头 (LEARNER_VISIBLE_METADATA_NOISE)，handson 指令清晰但页码混入。bridge 正确指向下一节"Java 语言の程序代码"。
- **改写方向**: 移除页码前缀，以"概念→代码→输出"为主线重写 blocks；handson 改为"改 concept 变量值，观察哪一行输出变了"。

### GS2: intro-ch02-lesson-001 — 出力 (变量/类型/输出)

- **章节**: java-ch02, order 1/24
- **选择理由**: 学生第一次接触标准输出。第2章有 24 个 lesson (数量最多)，需要确保首节质量以建立章节基调。
- **当前质量观察**: 5 个 block 均以"教材18ページ"开头。handson 指令含页码。
- **改写方向**: 聚焦"System.out.println 的三个作用：输出、换行、调试"。

### GS3: intro-ch03-lesson-001 — 条件分岐 (条件分支)

- **章节**: java-ch03, order 1/42
- **选择理由**: 全课程 lesson 数最多的章节 (42 节)，v1 checker 在此章检测到大量重复内容，是后续改写的重点目标。
- **当前质量观察**: 5 blocks 以"教材44ページ"开头。bridge 指向"条件分支の例"。
- **改写方向**: 确保 42 节的条件分支教学覆盖 if、if-else、switch、嵌套等不同行为，每节代码行为有本质差异。

### GS4: intro-ch04-lesson-001 — メソッドとは (方法)

- **章节**: java-ch04, order 1/20
- **选择理由**: 方法/函数是 Java 编程的核心组织单元。本章承上启下（前有变量和分支，后有类和对象）。
- **改写方向**: 区分"方法的定义"和"方法的调用"，handson 指引学生定义一个新方法并调用它。

### GS5: intro-ch05-lesson-001 — 实例化 (class / constructor)

- **章节**: java-ch05, order 1/18
- **选择理由**: 面向对象编程的入口。需要帮助学生建立"类=模板，实例=具体对象"的心智模型。
- **改写方向**: 代码示例用 StudentCard 类（已存在），强调"new"关键字的行为。

### GS6: intro-ch06-lesson-001 — コンストラクタ (构造方法)

- **章节**: java-ch06, order 1/16
- **选择理由**: 构造方法是 Java 中容易混淆的概念（与普通方法的区别）。本章的 StudentCard 示例是 Java 教学经典。
- **改写方向**: 对比"构造方法 vs 普通方法"的调用时机和返回值差异。

### GS7: intro-ch07-lesson-001 — 継承とは (继承)

- **章节**: java-ch07, order 1/22
- **选择理由**: 继承是面向对象三大特性之一。本章是 intro 阶段的最后一个概念性章节。
- **改写方向**: 从"复用代码"角度引入继承，用父类→子类的具体例子说明 extends 的行为。

### GS8: practice-ch04-lesson-001 — System.gc() (垃圾回收 / GC)

- **章节**: java-ch12 (practice-ch04), order 1/8
- **选择理由**: 本章包含 `System.gc()` 的合法教学使用，是 v2 checker 的 TEMP E 测试场景。仅 8 个 lesson，适合作为小批量改写的试点。
- **改写方向**: 保留 `System.gc()` 的教学内容，同时确保解释这不是"魔法命令"而是"建议请求"。

## 5. 后续改写安全路线

1. **每轮只改写 1 个 chapter（最多 42 节，最少 5 节）**
2. **改写顺序按黄金样本优先级**：GS1 (ch01) → GS3 (ch03) → GS8 (ch12) → GS2 (ch02) → GS4-7 → 其余章节
3. **每轮改写后必须通过**：
   - v2 checker (内容完整性)
   - 现有 R7 checker 套件 (source coverage, bilingual quality, no-quiz, runtime package, examples compile, route shell)
   - 新增 regression: 改写前后代码行为变化对比
4. **每轮 commit 独立**：一个 chapter = 一个 commit，可独立 revert
5. **不使用批量脚本**：改写由人工（或单个非并行 Agent）逐节完成，确保每节有独有内容
6. **改写前从本节 v2 聚类库存中提取模板化部分**，改写后验证相似度下降

## 6. 禁止的改写方法（永久黑名单）

以下方法因在 146df81/791637e 及脏工作区中产生不合格内容，永久禁止：

1. **按 lessonId 序号批量 spawn subprocess 写入** (`update_runner.js` 模式)
2. **按字段名直接替换字符串** (`update_lesson.js` 模式)
3. **先全局替换再检查器补救** (`fix_repeated.js` 模式)
4. **通过标题/页码/变量名/class 名/日志文字制造差异** (模板化反模式)
5. **注释掉 checker 门禁以求通过** (146df81 反模式)
6. **使用 semanticFidelity / profile / pedagogicalDelta 等布尔标记自我证明质量** (自欺欺人模式)
