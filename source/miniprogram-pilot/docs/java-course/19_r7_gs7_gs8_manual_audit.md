### 特别技术审计说明

#### 1. 为什么 GS7 继承不能使用 has-a 关系？
- **is-a 关系是继承的本质**：继承必须符合“子类是父类的一种特例”这一逻辑（如 `Student` is a `Person`）。
- **has-a 关系代表所有权/关联**：若逻辑上为“学生拥有包（`Student` has a `Bag`）”，应当使用组合（Composition，即将 `Bag` 实例作为 `Student` 的字段），而非通过继承使 `Student` 成为 `Bag` 的子类。滥用继承会导致类关系混乱并引发强耦合。

#### 2. 为什么 GS8 的程序逻辑绝对不可依赖 GC 时机？
- **GC 的非确定性行为**：JVM 垃圾回收器的运行受复杂的堆状态、GC 算法以及线程调度等诸多不可预测因素影响，`System.gc()` 也仅仅是“推荐执行”而非强制触发。
- **潜在风险**：如果程序的后续行为（如释放外部资源、依赖对象状态更新等）假定在 `System.gc()` 调用后的下一行一定完成回收，那么当 JVM 延迟或忽略 GC 请求时，程序就会产生随机性的资源泄漏、运行错误或崩溃。因此，程序逻辑的设计必须基于对象的解引（如 `null`）与状态的显式生命周期管理，决不能基于 GC 执行的微秒级猜测。

#### 3. 与 GS1-GS6 的差异性
- 本批次课程均使用完全新颖的实际业务代码示例（GS7 为 `Person-Student` 继承，GS8 为 `SampleObject` 内存分配与引用解除），且其 terms、Mistakes 与 Hands-on 完全独立开发，拒绝套用此前章节在集合、方法或变量上的教学模板，保证了课程的多样性与高标准。
